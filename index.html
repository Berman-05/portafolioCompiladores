01 — Introducción

¿Qué es el RPG Script Compiler?

Un compilador desarrollado en Python para validar y procesar scripts de videojuegos de rol. Permite definir personajes, habilidades y estados de forma estructurada, garantizando la corrección del código antes de su ejecución.

02 — Descripción del Lenguaje

Sintaxis del Lenguaje RPG

El lenguaje usa bloques delimitados por llaves con declaraciones tipo clave-valor. Cada entidad se define con una palabra reservada, un identificador y sus propiedades.

ejemplo.rpg personaje Heroe { HP = 100; MP = 50; } habilidad Golpe { daño = 25; } estado Quemado { efecto = "pierde vida"; }
Bloques { }
Cada entidad se delimita con llaves { }, definiendo su contexto y sus propiedades internas.
Asignaciones con =
Las propiedades se asignan con = seguido de un valor numérico o cadena.
Punto y coma obligatorio
Cada declaración termina con ; para marcar el fin de instrucción de forma explícita.
Tipos de datos
Enteros (100, 25) · Cadenas ("pierde vida") · Identificadores (Heroe)

04 — Analizador Sintáctico

Validación de Estructura

Verifica que la secuencia de tokens sigue las reglas gramaticales del lenguaje. Si la estructura no es válida, se reportan errores con la ubicación exacta.

Estructura válida
personaje Heroe { HP = 100; }
Estructura inválida
personaje Heroe // Error: falta llave de apertura { HP = 100;
Reglas Gramaticales
Todo bloque abre con { y cierra con }
Cada propiedad termina obligatoriamente con ;
Debe existir un identificador tras la palabra reservada
El valor de cada propiedad debe ser número o cadena
El operador = es obligatorio en cada asignación
Árbol Sintáctico (AST)
Programa ├── personaje │ └── Heroe │ ├── HP = 100 │ └── MP = 50 ├── habilidad │ └── Golpe │ └── daño = 25 └── estado └── Quemado └── efecto = "pierde vida"
FASE 03
05 — Analizador Semántico

Validación Lógica

Verifica que el código tiene sentido lógico: identificadores declarados, propiedades válidas para cada entidad y tipos de datos correctos.

E01
Identificador no declarado
Usar Dragon sin haberlo declarado con personaje
E02
Identificador duplicado
Declarar personaje Heroe dos veces en el mismo script
E03
Asignación inválida
HP = "texto" — HP espera un valor numérico
E04
Propiedad no válida
Usar efecto dentro de un bloque personaje — solo aplica a estados
E05
Tipo incorrecto
daño = "fuerte" — daño debe ser un entero
E06
Propiedad duplicada
Declarar HP dos veces dentro del mismo bloque Heroe
E07
Propiedad fuera de contexto
Colocar XP dentro de habilidad — XP pertenece solo a personaje

07 — Resultado Final

Salidas del Compilador

Una vez procesado el script, el compilador produce cuatro tipos de salida para verificar completamente el estado del código.

Lista de Tokens
Secuencia completa de tokens con tipo, valor y número de línea, generada durante el análisis léxico.
Árbol Sintáctico
Representación jerárquica de la estructura del programa con sus entidades, bloques y propiedades.
Tabla de Símbolos
Registro de todas las entidades declaradas con sus categorías, atributos y tipos de datos.
Lista de Errores
Reporte de errores léxicos, sintácticos y semánticos con código, descripción y línea de ocurrencia.
